class occupancy_grid.Cell¶

Class that represents cell of occupancy grid with lidar data.

fit(points, pos, over, neg, labels=None)¶

Fits cell’s state by given points and thresholds.

Parameters:
  • points (numpy.ndarray) – array n×3 with coordinates of points
  • pos (float) – threshold for positive obstacles
  • over (float) – threshold for overhanging structures
  • neg (float) – threshold for negaative obstacles
  • labels (list) – list of labels if points were classified before, e. g. with ground/non-ground segmentation

Return type:float

Simple getter of cell’s state.

State can be occupied (true) or free (false).

Returns:state
Return type:bool
set_occupancy(val)¶

Setter of cell’s occupancy value.

Occupancy is calculated as max() of occupancy and val.

Parameters:val (float) – given value of occupancy.

Warning

Occupancy value must be in [0, 1]!

stereo_matcher.py¶

class stereo_matcher.StereoToPointCloud(left, right, path_to_calib, key='stereo23', to='velo', range_filter=200)¶

Class that converts two rectified images from stereo to 3-dimensional point cloud.

Note

Stereo matching algorithm and it’s parameters are hardcoded!

Initialize paths to data directories and creates RT matrix from key to to coordinate system.

Parameters:
  • left (str) – path to directory with left stereo images
  • right (str) – path to directory with right stereo images
  • path_to_calib (str) – path to KITTI calibration files: calib_cam_to_cam.txt, calib_imu_to_velo.txt, calib_velo_to_cam.txt
  • key (str) – initial coordinate system
  • to (str) – final coordinate system
  • range_filter (float) – range for filtering points
get_point_cloud(ind)¶

Converts to images to 3D point cloud.

Returns:img – array n×3 with coordinates of points,

clr – array n×3 with RGB colors of points
